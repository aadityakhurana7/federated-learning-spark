# 🚀 Federated Learning Optimization for Large-Scale Distributed Systems using Apache Spark

This project demonstrates a highly optimized, distributed **Federated Learning** system built with **Apache Spark (PySpark)**. It solves the massive network congestion and privacy risks associated with traditional centralized machine learning by training models locally on distributed worker nodes and securely aggregating only the mathematical model weights.

## 🌟 Key Features (Big Data Concepts Implemented)
1. **Parallel Processing:** Uses Spark RDDs (`rdd.map`) to compute machine learning models on multiple worker nodes at the exact same time.
2. **Data Locality & Caching:** The dataset is partitioned across the Spark cluster and cached in memory (`.cache()`). Raw data never leaves the local worker node.
3. **Distributed Aggregation:** Utilizes Spark's `rdd.reduce()` to safely aggregate model coefficients in parallel, preventing bottlenecks on the master node.
4. **Communication Optimization (99.9% Savings):** Uses `sc.broadcast()` to send global models. We transfer only ~168 Bytes of model weights per round instead of 1.2+ MB of raw data.
5. **Fault Tolerance:** Implements Spark checkpointing and physical `.pkl` model checkpoints every 5 rounds to prevent data loss upon node failure.
6. **Strict Data Privacy:** A `PrivacyEnforcer` middleware guarantees raw data never leaks and injects **Differential Privacy (Gaussian Noise)** into the weights to prevent reverse-engineering.

## 💻 How to Run This Project on a New Laptop

Follow these exact steps to clone the project, install the dependencies, and run the live Dashboard on any new machine.

### Prerequisites
- Python 3.9 or higher installed
- Git installed

### Step 1: Clone the Repository
Open a terminal on the new laptop and run:
```bash
git clone https://github.com/aadityakhurana7/federated-learning-spark.git
cd federated-learning-spark
```

### Step 2: Create a Virtual Environment & Install Dependencies
It is highly recommended to use a virtual environment so you don't mess up your system Python packages.
```bash
# Create a virtual environment named 'venv'
python3 -m venv venv

# Activate the virtual environment (Mac/Linux)
source venv/bin/activate
# OR if you are on Windows:
# venv\Scripts\activate

# Install the required packages (PySpark, Streamlit, Scikit-learn, etc.)
pip install -r requirements.txt
```

### Step 3: Run the Federated Learning Training (Spark)
This will initialize the local Spark cluster, generate the distributed dataset, and run the FedAvg algorithm for 10 rounds.
```bash
# Make sure your virtual environment is still activated!
python -m src.main
```
*(When it finishes, it will pause and give you a link to `http://localhost:4040` to view the raw Apache Spark DAGs and memory storage).*

### Step 4: Launch the Real-Time Dashboard
Open a **new** terminal tab, activate the environment again, and run the dashboard:
```bash
cd federated-learning-spark
source venv/bin/activate
streamlit run dashboard/app.py
```
This will automatically open `http://localhost:8501` in your browser where you can see the ML Convergence comparisons, Data Locality pie charts, and Network Savings calculations!

## 📁 Project Architecture
```text
federated_spark/
├── requirements.txt            # Python dependencies
├── src/
│   ├── config.py               # Global hyperparams (epochs, clients, learning rate)
│   ├── spark_manager.py        # SparkSession and Checkpoint initialization
│   ├── data_loader.py          # Generates and partitions the dataset for workers
│   ├── client.py               # Local ML training & Differential Privacy enforcer
│   ├── server.py               # FedAvg Server logic
│   ├── centralized_baseline.py # Centralized ML trainer (for baseline comparison)
│   └── main.py                 # Core Orchestrator (Broadcasts, Maps, Reduces)
├── data/
│   └── synthetic_dataset.csv   # The physical 10,000 row dataset generated by the script
├── logs/
│   ├── metrics.json            # Dynamic output read by Streamlit
│   ├── baseline_metrics.json   # Output from the centralized baseline
│   └── spark_checkpoints/      # Fault tolerance backups
└── dashboard/
    └── app.py                  # Streamlit frontend with Big Data visual proofs
```
